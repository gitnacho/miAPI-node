Miapi-node

Mi interpretación de la API de Node v0.8.8

TTY

Estabilidad: 2 — Inestable

El módulo tty alberga las clases tty.ReadStream y tty.WriteStream. En la mayoría de los casos, no tendrás que utilizar este módulo directamente.

Cuando Node detecta que se está ejecutando dentro de un contexto TTY, entonces process.stdin será una instancia de tty.ReadStream y process.stdout será una instancia de tty.WriteStream. La mejor forma de comprobar si Node se está ejecutando en un contexto TTY es revisando process.stdout.isTTY:

$ node -p -e 'Boolean(process.stdout.isTTY)'
true
$ node -p -e 'Boolean(process.stdout.isTTY)' | cat
false

tty.isatty(descriptorArchivo)

Devuelve true o false dependiendo de si el descriptorArchivo está asociado con una terminal.

tty.setRawMode(modo)

Desaconsejado. En su lugar usa tty.ReadStream#setRawMode() (es decir, process.stdin.setRawMode()).

Clase: ReadStream

Una subclase de net.Socket que representa la parte legible de un TTY. En circunstancias normales, process.stdin será la única instancia de tty.ReadStream, en cualquier programa Node (sólo cuando isatty(0) es true).

rs.isRaw

Un Booleano que se inicia en false. Representa el estado actual «sin procesar» de la instancia tty.ReadStream.

rs.setRawMode(modo)

modo debe ser true o false. Este establece las propiedades del tty.ReadStream para actuar ya sea como un dispositivo sin procesar o el predefinido. isRaw se fijará al modo resultante.

Clase: WriteStream

Una subclase de net.Socket que representa la parte de escritura de un tty. En circunstancias normales, process.stdout será la única instancia de tty.WriteStream jamás creada (y sólo cuando isatty(1) es true).

ws.columns

Un Número que proporciona la cantidad de columnas que la tty tiene actualmente. Esta propiedad se actualiza en los eventos 'resize'.

ws.rows

Un Número que proporciona la cantidad de filas que la tty tiene actualmente. Esta propiedad se actualiza en los eventos 'resize'.

Evento: 'resize'

function () {}

Emitido por refreshSize() cuando cualquiera de las propiedades columns o rows ha cambiado.

process.stdout.on('resize', function () {
  console.log('¡el tamaño de la pantalla ha cambiado!');
  console.log(process.stdout.columns + 'x' + process.stdout.rows);
});
