    Sobre esta documentación

    El objetivo de este documento es explicar integralmente la API de Node.js, tanto desde un punto de vista conceptual como de referencia. Cada sección describe un módulo integrado o un concepto de alto nivel.

    Dónde es apropiado, los tipos de las propiedades, argumentos de método y argumentos proporcionados a los controladores de eventos se detallan en una lista debajo del título del tema.

    Cada documento .html tiene su documento .json correspondiente que presenta la misma información de una manera estructurada. Esta característica es experimental y se agregó para beneficio del IDE y otros servicios públicos que desean hacer cosas programáticamente con la documentación.

    Cada archivo .html y .json se genera en base al archivo .markdown correspondiente en el directorio doc/api/ del árbol de directorios fuente de Node. La documentación se genera utilizando el programa tools/doc/generate.js. La plantilla HTML se encuentra en doc/template.html.

    Índice de estabilidad

    A lo largo de la documentación, verás las indicaciones de la estabilidad de una sección. La API de Node.js todavía es algo cambiante y a medida que madura, ciertas partes son más confiables que otras. Algunas están bastante probadas y se basan en lo que es poco probable que cambie en algún momento. Otras son completamente nuevas y experimentales, o se considera que son peligrosas y están en proceso de rediseño.

    Los avisos tienen este aspecto:

    Estabilidad: 1 — Experimental

    Los índices de estabilidad son los siguientes:

    • 0 — Desaconsejada Esta característica se sabe que es problemática y se han previsto cambios. No confíes en ella. El uso de la característica puede provocar advertencias. No debes esperar compatibilidad con versiones anteriores.

    • 1 — Experimental Esta característica se introdujo recientemente y puede cambiar o eliminarse en futuras versiones. Por favor, pruébala y proporciona retroalimentación. Si esta aborda un caso de uso que es importante para ti, díselo al equipo del núcleo de Node.

    • 2 — Inestable La API se encuentra en el proceso de asentamiento, pero todavía no ha sido suficientemente probada en el mundo real para que se considere estable. Se mantendrá la compatibilidad con versiones anteriores, si es razonable.

    • 3 — Estable La API ha resultado satisfactoria, pero la limpieza en el código subyacente puede provocar cambios menores. La compatibilidad con versiones anteriores está garantizada.

    • 4 — API congelada Esta API ha sido probada ampliamente en producción y es poco probable que alguna vez cambie.

    • 5 — Cerrada A menos que se encuentren errores graves, este código nunca cambiará. Por favor, no sugieras cambios en esta área; porque serán rechazados.

    Eventos

    Estabilidad: 4 — API congelada

    En Node muchos objetos emiten eventos: un net.Server emite un evento al abrir un archivo. Todos los objetos que emiten eventos son instancias de events.EventEmitter. Puedes acceder a este módulo invocando a: 

    var events = require('events');

    Típicamente, los nombres de evento están representados por una cadena con mayúsculas intercaladas, sin embargo, no hay restricciones estrictas en ellas, dado que cualquier cadena será aceptada.

    Puedes adjuntar funciones a objetos, para ser ejecutadas cuando se emita un evento. Estas funciones se conocen como escuchas.

    Clase: events.EventEmitter

    Para acceder a la clase EventEmitter usa:

    var emisor = require('events').EventEmitter

    Cuando una instancia de EventEmitter experimenta un error, la acción típica consiste en emitir un evento 'error'. En Node, los eventos de error se tratan como un caso especial. Si no hay escucha para él, entonces la acción predeterminada es imprimir el seguimiento de la pila y salir del programa.

    Todos los escuchas.

    emisor.addListener(evento,escucha)

    emisor.on(evento,escucha)

    Añade un escucha al final del arreglo de escuchas del evento especificado.

    servidor.on('connection', function (flujo) {
  console.log('¡alguien se ha conectado!');
});

    emisor.once(evento,escucha)

    Agrega un escucha fugaz para el evento. Este escucha, es invocado únicamente la siguiente vez que se emita el evento, tras la cual se elimina.

    servidor.once('connection', function (flujo) {
  console.log('¡Ah, tenemos nuestro primer usuario!');
});

    emisor.removeListener(evento,escucha)

    Quita un evento especificado. Precaución: esta cambia los índices del arreglo en la matriz de escuchas detrás del escucha.

    var retrollamada = function (flujo) {
  console.log('¡alguien se ha conectado!');
};
server.on('connection', retrollamada);
// ...
server.removeListener('connection', retrollamada);

    emisor.removeAllListeners([evento])

    Quita todos los escuchas, o los relacionados al evento especificado.

    Ten en cuenta que esto invalidará algunos arreglos que anteriormente fueron devueltos por emisor.listeners(evento).

    emisor.setMaxListeners(cantidad)

    De manera predeterminada EventEmitters imprimirá una notificación si se añaden más de 10 escuchas a un evento en particular. Se trata de una utilidad predeterminada que te ayuda a encontrar fugas de memoria. Obviamente, no todos los emisores se deben limitar a 10. Esta función te permite incrementarlos. Ajústala a cero para ilimitados.

    emisor.listeners(evento)

    Devuelve un evento especificado.

    servidor.on('connection', function (flujo) {
  console.log('¡alguien se ha conectado!');
});
console.log(util.inspect(servidor.listeners('connection'))); // [ [Función] ]

    Este arreglo puede ser una referencia mutable a la misma lista de escuchas subyacente que utiliza el subsistema de eventos. Sin embargo, ciertas acciones (específicamente, removeAllListeners) invalidarán esta referencia.

    Si quieres obtener una copia de los escuchas en un punto específico en el tiempo que te garantice que no cambiará, haz una copia, por ejemplo haciendo emisor.listeners(evento).slice(0).

    En una futura versión de Node, este comportamiento puede cambiar para siempre devolver una copia, por consistencia. En tus programas, por favor, no confíes en poder modificar los escuchas de EventEmitter utilizando métodos de arreglo. Siempre utiliza el método 'on' para añadir nuevos escuchas.

    emisor.emit(evento, [arg1], [arg2], [...])

    Ejecuta cada uno de los escuchas en orden con los argumentos proporcionados.

    Evento: 'newListener'

    • evento {Cadena} El nombre del evento
    • escucha {Función} La función controladora del evento

    Este evento se emite al momento de agregar un nuevo escucha.
