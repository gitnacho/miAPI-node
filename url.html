URL

Estabilidad: 3 — Estable

Este módulo tiene utilidades para la resolución y análisis de direcciones URL. Para acceder a este módulo usa:

var url = require('url');

Los objetos URL analizados tienen alguno o todos los siguientes campos, en función de si existen o no en la cadena de la dirección URL. Las piezas que no están en la cadena de la dirección URL no estarán en el objeto analizado. Los ejemplos mostrados están basados en la dirección URL:

'http://usuario:pase@host.com:8080/r/u/t/a?consulta=cadena#codificada'

  • href: La dirección URL completa analizada en un principio. Tanto el protocolo como el servidor están en minúsculas.

    Ejemplo: 'http://usuario:pase@host.com:8080/r/u/t/a?consulta=cadena#codificada'

  • protocol: El protocolo de la petición, en minúsculas.

    Ejemplo: 'http:'

  • host: La porción completa del servidor en minúsculas de la URL, incluyendo información del puerto.

    Ejemplo: 'host.com:8080'

  • auth: La parte de la información de autenticación de una URL.

    Ejemplo: 'usuario:pase'

  • hostname: Sólo la porción en minúsculas del nombre de la máquina.

    Ejemplo: 'host.com'

  • port: La parte correspondiente al número de puerto del servidor.

    Ejemplo: '8080'

  • pathname: La sección de ruta de la URL, la cual viene después del nombre de la máquina y antes de la cadena de consulta, incluyendo la barra inclinada inicial si está presente.

    Ejemplo: '/r/u/t/a'

  • search: La parte de la «cadena de consulta» de la URL, incluyendo el signo de interrogación inicial.

    Ejemplo: '?consulta=cadena'

  • ruta: La concatenación de pathname y search.

    Ejemplo: '/r/u/t/a?consulta=cadena'

  • query: O bien la parte 'params' de la cadena de consulta, o un objeto cadenaDeConsulta analizado.

    Ejemplo: 'consulta=cadena' o { 'consulta': 'cadena' }

  • hash: La porción 'fragmento' de la dirección URL incluido el signo de número.

    Ejemplo: '#codificado'

El módulo URL proporciona los siguientes métodos:

url.parse(cadenaUrl, [cadenaDeConsultaPorAnalizar], [barrasDenotanServidor])

Toma una cadena de dirección URL, y devuelve un objeto.

Pasa true como segundo argumento para analizar también la cadena de consulta utilizando el módulo de cadenas de consulta, querystring. De manera predeterminada es false.

Pasa true como tercer argumento para tratar //loquesea/bar como { host: 'loquesea', pathname: '/bar' } en lugar de { pathname: '//loquesea/bar' }. De manera predeterminada es false.

url.format(objetoUrl)

Toma un objeto URL analizado y devuelve una cadena con formato URL.

  • href será descartado.
  • protocol es tratado como el mismo con o sin los : (dos puntos) finales.
    • Los protocolos http, https, ftp, gopher, file se deben posfijar con `://' (dos puntos-barra inclinada-barra inclinada).
    • Todos los demás protocolos mailto, xmpp, aim, sftp, loquesea, etc. se deben posfijar con dos puntos (:).
  • auth se utilizará si está presente.
  • hostname sólo se utilizará si host está ausente.auth se utilizará si está presente.
  • port sólo se utilizará si host está ausente.
  • host se utiliza en lugar de nombreServidor y puerto
  • pathname se trata igual con o sin la / (barra inclinada)
  • search se utiliza en lugar de query
  • query (objeto; consulta cadenaDeConsulta) sólo se utilizará si search está ausente.
  • search es tratada igual con o sin el signo de interrogación inicial ?
  • hash se trata igual con o sin el # (signo de número, ancla)

url.resolve(desde, hasta)

Toma una dirección URL base y una dirección href URL, y las resuelve como una ancla del navegador.
