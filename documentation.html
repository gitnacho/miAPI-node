Miapi-node

Mi interpretación de la API de Node v0.8.8

Sobre esta documentación

El objetivo de este documento es explicar integralmente la API de Node.js, tanto desde un punto de vista conceptual como de referencia. Cada sección describe un módulo integrado o un concepto de alto nivel.

Dónde es apropiado, los tipos de las propiedades, argumentos de método y argumentos proporcionados a los controladores de eventos se detallan en una lista debajo del título del tema.

Cada documento .html tiene su documento .json correspondiente que presenta la misma información de una manera estructurada. Esta característica es experimental y se agregó para beneficio del IDE y otros servicios públicos que desean hacer cosas programáticamente con la documentación.

Cada archivo .html y .json se genera en base al archivo .markdown correspondiente en el directorio doc/api/ del árbol de directorios fuente de Node. La documentación se genera utilizando el programa tools/doc/generate.js. La plantilla HTML se encuentra en doc/template.html.

Índice de estabilidad

A lo largo de la documentación, verás las indicaciones de la estabilidad de una sección. La API de Node.js todavía es algo cambiante y a medida que madura, ciertas partes son más confiables que otras. Algunas están bastante probadas y se basan en lo que es poco probable que cambie en algún momento. Otras son completamente nuevas y experimentales, o se considera que son peligrosas y están en proceso de rediseño.

Los avisos tienen este aspecto:

Estabilidad: 1 — Experimental

Los índices de estabilidad son los siguientes:

  • 0 — Desaconsejada Esta característica se sabe que es problemática y se han previsto cambios. No confíes en ella. El uso de la característica puede provocar advertencias. No debes esperar compatibilidad con versiones anteriores.

  • 1 — Experimental Esta característica se introdujo recientemente y puede cambiar o eliminarse en futuras versiones. Por favor, pruébala y proporciona retroalimentación. Si esta aborda un caso de uso que es importante para ti, díselo al equipo del núcleo de Node.

  • 2 — Inestable La API se encuentra en el proceso de asentamiento, pero todavía no ha sido suficientemente probada en el mundo real para que se considere estable. Se mantendrá la compatibilidad con versiones anteriores, si es razonable.

  • 3 — Estable La API ha resultado satisfactoria, pero la limpieza en el código subyacente puede provocar cambios menores. La compatibilidad con versiones anteriores está garantizada.

  • 4 — API congelada Esta API ha sido probada ampliamente en producción y es poco probable que alguna vez cambie.

  • 5 — Cerrada A menos que se encuentren errores graves, este código nunca cambiará. Por favor, no sugieras cambios en esta área; porque serán rechazados.
